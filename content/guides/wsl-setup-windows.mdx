# WSL Development Setup (Windows)

Windows 환경에서 **WSL + Ubuntu + VS Code + GitHub SSH** 기반 개발 환경을 구축하는 과정을 정리한 가이드입니다.

이 문서는 다음 흐름을 기준으로 작성했습니다.

- WSL 설치
- Ubuntu 초기 설정
- 기본 개발 패키지 설치
- Node.js 설치
- VS Code WSL 연결
- GitHub SSH 인증
- Git remote를 SSH로 전환

---

## 1. WSL 설치

PowerShell을 **관리자 권한**으로 실행한 뒤 아래 명령을 입력합니다.

```bash
wsl --install
```

설치가 완료되면 PC를 재부팅합니다.

---

## 2. Ubuntu 설치

WSL에서 Ubuntu를 설치합니다.

```bash
wsl --install -d Ubuntu
```

설치가 완료되면 Ubuntu가 시작되며, 기본 사용자 계정을 생성하게 됩니다.

예시:

```text
Create a default Unix user account:
```

여기서 원하는 사용자 이름과 비밀번호를 입력하면 됩니다.

---

## 3. 패키지 업데이트

Ubuntu 초기 설정이 끝나면 패키지를 최신 상태로 업데이트합니다.

```bash
sudo apt update
sudo apt upgrade -y
```

---

## 4. 개발 기본 패키지 설치

개발에 필요한 기본 패키지를 설치합니다.

```bash
sudo apt install -y git curl build-essential
```

설치 확인:

```bash
git --version
curl --version
gcc --version
```

---

## 5. 프로젝트 폴더 생성

WSL에서는 **Windows 경로보다 Linux 경로에서 작업하는 것이 좋습니다.**

먼저 홈 디렉터리 아래에 프로젝트 폴더를 만듭니다.

```bash
cd ~
mkdir -p ~/projects
cd ~/projects
```

현재 위치 확인:

```bash
pwd
```

예상 결과:

```text
/home/username/projects
```

---

## 6. Node.js 설치 (NVM)

Node Version Manager를 설치합니다.

```bash
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.7/install.sh | bash
```

설치 후 bash 설정을 다시 불러옵니다.

```bash
source ~/.bashrc
```

설치 확인:

```bash
nvm --version
```

이후 LTS 버전의 Node.js를 설치합니다.

```bash
nvm install --lts
```

설치 확인:

```bash
node -v
npm -v
```

---

## 7. VS Code에서 WSL 연결

Windows에 **Visual Studio Code**를 설치합니다.

그 다음 Extensions에서 아래 확장을 설치합니다.

```text
WSL
```

설치 후 WSL 터미널에서 프로젝트 폴더로 이동해 VS Code를 실행합니다.

```bash
cd ~/projects
code .
```

정상적으로 연결되면 VS Code 좌측 하단에 아래와 같은 표시가 보입니다.

```text
WSL: Ubuntu
```

---

## 8. GitHub SSH 키 생성

회사 노트북이나 새 PC에서는 **기기별로 별도 SSH 키를 생성하는 것이 정석**입니다.

아래 명령으로 SSH 키를 생성합니다.

```bash
ssh-keygen -t ed25519 -C "moritaseo1004@github"
```

질문이 나오면 기본 경로는 그냥 엔터를 눌러 진행하면 됩니다.

예시:

```text
Enter file in which to save the key (/home/username/.ssh/id_ed25519):
```

passphrase는 보안을 위해 설정해도 되고, 편의를 위해 비워도 됩니다.

---

## 9. SSH Agent 실행 및 키 등록

SSH Agent를 실행합니다.

```bash
eval "$(ssh-agent -s)"
```

생성한 키를 등록합니다.

```bash
ssh-add ~/.ssh/id_ed25519
```

공개키 확인:

```bash
cat ~/.ssh/id_ed25519.pub
```

출력된 문자열 전체를 복사해서 GitHub에 등록합니다.

경로:

```text
GitHub → Settings → SSH and GPG keys → New SSH key
```

추천 입력값:

- Title: `Company Laptop WSL`
- Key type: `Authentication Key`

---

## 10. SSH 연결 테스트

GitHub와의 SSH 연결을 확인합니다.

```bash
ssh -T git@github.com
```

처음 연결 시 아래와 같은 메시지가 나올 수 있습니다.

```text
Are you sure you want to continue connecting (yes/no/[fingerprint])?
```

여기서 `yes`를 입력합니다.

정상 결과 예시:

```text
Hi moritaseo1004! You've successfully authenticated, but GitHub does not provide shell access.
```

이 메시지가 보이면 SSH 인증은 성공입니다.

---

## 11. Git Remote를 SSH로 변경

기존 repo가 HTTPS로 연결되어 있다면 SSH 주소로 변경합니다.

현재 remote 확인:

```bash
git remote -v
```

예시:

```text
origin  https://github.com/moritaseo1004/morita-vibe-coding-journey.git (fetch)
origin  https://github.com/moritaseo1004/morita-vibe-coding-journey.git (push)
```

SSH로 변경:

```bash
git remote set-url origin git@github.com:moritaseo1004/morita-vibe-coding-journey.git
```

다시 확인:

```bash
git remote -v
```

정상 결과:

```text
origin  git@github.com:moritaseo1004/morita-vibe-coding-journey.git (fetch)
origin  git@github.com:moritaseo1004/morita-vibe-coding-journey.git (push)
```

---

## 12. Git 동기화 테스트

SSH 전환 후 pull이 정상적으로 동작하는지 확인합니다.

```bash
git pull origin main
```

예상 결과:

```text
Already up to date.
```

이 상태라면 현재 로컬 프로젝트와 GitHub 저장소가 정상적으로 연결된 것입니다.

---

## 13. VS Code에서 패키지 설치 및 개발 서버 실행

프로젝트를 새 PC에서 처음 clone 했다면 `node_modules`가 없기 때문에 아래 명령을 먼저 실행합니다.

```bash
npm install
```

그 다음 개발 서버를 실행합니다.

```bash
npm run dev
```

브라우저에서 아래 주소로 확인할 수 있습니다.

```text
http://localhost:3000
```

---

## 14. 추천 작업 흐름

새 PC에서 프로젝트를 이어서 작업할 때 기본 흐름은 다음과 같습니다.

```bash
cd ~/projects
git clone git@github.com:moritaseo1004/morita-vibe-coding-journey.git
cd morita-vibe-coding-journey
npm install
code .
npm run dev
```

작업 후에는 아래 흐름으로 반영합니다.

```bash
git add .
git commit -m "Add WSL setup guide"
git push
```

## Tips

### Linux 경로에서 작업하기

WSL에서는 다음 경로를 권장합니다.

```text
/home/username/projects
```

반대로 아래 경로는 개발 작업 위치로 권장하지 않습니다.

```text
/mnt/c/Users/...
```

### 새 기기마다 SSH 키를 별도로 생성하기

집 PC와 회사 노트북은 **각각 다른 SSH 키**를 GitHub에 등록하는 것이 좋습니다.

예를 들면 GitHub의 SSH Keys 목록이 아래처럼 구성될 수 있습니다.

```text
Home Desktop
Company Laptop
MacBook
```

이렇게 하면 특정 기기만 선택적으로 제거할 수 있어서 보안 관리가 편합니다.
